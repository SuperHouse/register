These filters are applied, in order, when importing parts from a BOM CSV: a row is first checked against the Supplemental Part Rules, which may add extra rows alongside it; every resulting row (the original and any supplements) is then checked against the Exclusion Rules; if not excluded, the Transformation Rules are applied to remap its Library, Device, Package, and/or Value; finally, the Library Settings determine whether the Device, Package, and/or Value fields are ignored for that row's library.

1. Supplemental Part Rules

Some footprints imply a second, separately-stocked part that never gets its own line in the BOM export - e.g. a fuse holder footprint whose Value ("1A", "2A", ...) really means the board also needs a fuse of that rating. A row that matches every non-blank Library/Device/Package/Value field on a rule has an extra row added alongside it, built from the Supplement fields (leave a Supplement field blank to copy the matched row's value for it - this is what lets a fuse holder's Value carry straight through to the generated fuse row without a separate rule per rating). The new row's reference designator is the matched row's reference with Reference Suffix appended (e.g. "-FUSE" turns F1 into F1-FUSE), and it is then excluded/transformed/ imported exactly like any other row. If a Library Setting below has Ignore Value checked for the matched library (e.g. to collapse a fuse holder to one Part regardless of rating), give Supplement Library a distinct name rather than leaving it blank - otherwise that same setting also blanks the generated row's Value.

3. Transformation Rules

Rows that match every non-blank "From" field on a rule have their Library, Device, Package, and/or Value replaced with the rule's "To" values before being checked for duplicates and imported. Leave a "From" field blank to match any value for that field; leave a "To" field blank to leave it unchanged.
